## 一、 核心网站模块划分 (Website Modules)

为了保持页面简单、信息密度高，建议采用单页应用（SPA）**或者**双页（看板 + 个人详情弹窗/纯静态页）设计：

### 1. 顶部：实时战况总览 (Hero Stats)

* **总减重金腰带：** 醒目放大展示当前累计减重最多的人。
* **本周鸽王/自律王：** 展示本周缺卡最多和运动打卡最猛的人。
* **赛事倒计时/进度条：** 比如“第一届大赛已进行 1 星期，剩余 X 周”。

### 2. 中部：硬核龙虎榜 (Leaderboard Table)

* 一个支持排序的表格，包含以下字段：
* 排名 | 选手（昵称）| 初始体重 | 当前体重 | **累计减重 (kg)** | **减重百分比 (%)** | 本周出勤率 (X/7)


* **视觉整活：** 减重百分比最高的选手，条目背景闪烁金光；增重的选手（比如周导），体重数字直接变成红色并附带 📈 警告图标。

### 3. 下方：高能打卡日历墙 (Streak Grid)

* 类似 GitHub 贡献图的 7 列格子墙。
* 每一格代表一天，鼠标悬浮或点击可以弹窗显示当天的“人类观察日志”：
* 包含：*今日体重、运动、饮食、以及那句高能的心得（如：“比廖子强”、“墙角90度”）*。



### 4. 底部：AI 锐评茶话会 (AI Weekly Comments)

* 按周聚合的卡片区。
* 展示每周 AI 给每个人生成的毒舌/趣味评语，支持切换周数（第一周、第二周...）。

## 二、 极简技术实现方案 (Architecture & Workflow)

放弃数据库后，我们的核心技术栈和工作流可以精简为：**Vite + React + Tailwind CSS + 纯静态 JSON 数据**。

```
[ Excel / CSV 数据 ] 
       │
       ▼ (本地脚本自动化转换 / 手动粘贴)
[ src/data/history.json ] ──> [ React 静态打包 (Vite) ] ──> [ 免费托管 (GitHub Pages/Vercel) ]

```

### 1. 数据存储方案：全量 JSON

在项目的 `src/data/` 目录下放置一个 `history.json`（或直接写在 `.ts` 文件里），格式设计如下。每周只需要往这个数组里追加新的一周数据即可：

```json
[
  {
    "week": 1,
    "dateRange": "2026.06.01 - 2026.06.07",
    "participants": [
      {
        "uid": "Latrell",
        "height": 188,
        "initialWeight": 107,
        "currentWeight": 106,
        "aiComment": "开局一句“智取减肥王”气势磅礴……",
        "dailyRecords": [
          { "date": "2026-06-01", "weight": 107, "sport": "有氧+力量", "diet": "严格控制", "note": "智取减肥王" },
          { "date": "2026-06-02", "weight": 106.2, "sport": "有氧+力量", "diet": "严格控制", "note": "今天碳水后置多吃了一顿..." }
          // ... 7天数据
        ]
      }
      // ... 其他人
    ]
  }
]

```

### 2. 构建与更新流程（每周仅需 3 步）

* **Step 1：丢入数据**
每周比赛结束，把 Excel 的明细页和汇总页内容用个简单的 Python 脚本/Claude 直接转），生成上面那段最新的周 JSON 数据，追加到项目的 `history.json` 中。
* **Step 2：一键打包**
在本地终端运行 `npm run build`。React 会把所有 JSON 数据直接编译进静态的 HTML/JS 文件中，前端页面直接通过 `import data from './data.json'` 读取，**不需要任何 API 请求，秒开！**
* **Step 3：全量上线**
代码推送到 GitHub，通过 GitHub Actions 自动部署到 **GitHub Pages**（或者直接绑定 Vercel/Netlify，推上去 10 秒后网站就更新了）。
